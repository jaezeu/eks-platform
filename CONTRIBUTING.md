# Contributing to eks-platform

Thank you for your interest in contributing to this project! We welcome contributions of all kinds, including bug fixes, documentation improvements, new features, and examples.

## How to Contribute

1. **Fork the repository**
   - Click the "Fork" button at the top right of this page to create your own copy.

2. **Clone your fork**
   - `git clone https://github.com/jaezeu/eks-platform.git`

3. **Create a new branch**
   - Use a descriptive name, e.g. `feature/add-new-addon` or `fix/typo-in-readme`
   - `git checkout -b your-branch-name`

4. **Make your changes**
   - Follow the existing code style and structure.
   - Add or update documentation as needed.
   - If adding scripts or manifests, ensure they are well-commented and tested.

5. **Test your changes**
   - Run any relevant tests or validation steps.
   - Ensure your changes do not break existing functionality.

6. **Commit and push**
   - `git add .`
   - `git commit -m "Describe your changes"`
   - `git push origin your-branch-name`

7. **Open a Pull Request**
   - Go to the original repository and click "Compare & pull request".
   - Fill in a clear title and description of your changes.

8. **Respond to feedback**
   - Be ready to discuss and make changes if requested by reviewers.

## Guidelines

- Follow [Conventional Commits](https://www.conventionalcommits.org/) for commit messages.
- Keep pull requests focused and small; large changes are harder to review.
- Update documentation for any new features or changes.
- Respect the project's [LICENSE](LICENSE).

## Need Help?

If you have questions, open an issue or start a discussion in the repository.
